15 // 2. This generator should NOT be used as in the following line.
16 // for (int i = 0; i < 100; ++i) { RNG x; cout << x.uniform() << endl; }
17 // The problem is that each time through the loop, a new RNG 'x' is
18 // created, and that RNG is used to generate exactly one random number.
19 // While the results may be satisfactory, the class is designed to
20 // produce quality random numbers by having a single (or a few) RNGs
21 // called repeatedly.
22 // The better way to do the above loop is:
23 // RNG x; for (int i = 0; i < 100; ++i) { cout << x.uniform() << endl; }
24 
25 // __________________________________________________________________________
26 // This C++ code uses the simple, fast "KISS" (Keep It Simple Stupid)
27 // random number generator suggested by George Marsaglia in a Usenet
28 // posting from 1999. He describes it as "one of my favorite
29 // generators". It generates high-quality random numbers that
30 // apparently pass all commonly used tests for randomness. In fact, it
31 // generates random numbers by combining the results of three simple
32 // random number generators that have different periods and are
33 // constructed from completely different algorithms. It does not have
34 // the ultra-long period of some other generators - a "problem" that can
35 // be fixed fairly easily - but that seems to be its only potential
36 // problem. The period is about 2^123.
37 
38 // The KISS algorithm is only used directly in the function rand_int32.
39 // rand_int32 is then used (directly or indirectly) by every other
40 // member function of the class that generates random numbers. For
41 // faster random numbers, one can redefine rand_int32 to return either
42 // WMC(), CONG(), or SHR3(). The speed will be two to three times
43 // faster, and the quality of the random numbers should be sufficient
44 // for many purposes. The three alternatives are comparable in terms of
45 // both speed and quality.
46 
47 // The ziggurat method of Marsaglia is used to generate exponential and
48 // normal variates. The method as well as source code can be found in
49 // the article "The Ziggurat Method for Generating Random Variables" by
50 // Marsaglia and Tsang, Journal of Statistical Software 5, 2000.
51 
52 // The method for generating gamma variables appears in "A Simple Method
53 // for Generating Gamma Variables" by Marsaglia and Tsang, ACM
54 // Transactions on Mathematical Software, Vol. 26, No 3, Sep 2000, pages
55 // 363-372.
